# PRD 完整性自检清单

生成 PRD 前，逐项自检以下内容。全部通过后再开始生成文档。

## 逻辑闭环检查

- [ ] 每个用户旅程是否从头走到尾？有无断点或未定义的状态跳转？
- [ ] 每个功能的"谁 → 做什么 → 得到什么结果"链路是否完整？
- [ ] 核心流程中是否存在"然后呢？"无法回答的步骤？
- [ ] 异常路径是否覆盖？（正常流程能走通 ≠ 需求完整）

## 角色覆盖检查

- [ ] 是否遗漏了任何用户角色？（如管理员、审批人、运维等隐性角色）
- [ ] 每个角色的核心诉求是否都有对应功能支撑？
- [ ] 角色之间的权限边界是否清晰？

## 边界条件检查

- [ ] 最大值/最小值/空值情况是否考虑？
- [ ] 首次使用（无历史数据）的场景是否覆盖？
- [ ] 大量数据或高频操作时的产品行为是否明确？
- [ ] 并发操作时的产品预期是否明确？

## 依赖与联动检查

- [ ] 是否依赖其他系统/模块？依赖方的状态变化对本产品的影响是否考虑？
- [ ] 本产品的功能变化是否会波及其他模块？
- [ ] 数据从哪里来、到哪里去？

## 优先级合理性检查

- [ ] P0 功能去掉后产品是否还能成立？（如果不能，确认 P0 的合理性）
- [ ] P2 功能是否真的"锦上添花"而非"没有它流程跑不通"？

## 业务规则检查

- [ ] 每一条规则是否有触发条件和例外情况？
- [ ] 涉及计算/判断的规则是否能用公式或伪代码清晰表达？
- [ ] 规则之间是否存在矛盾或重复？

---

## 自检汇报格式

自检完成后，向用户汇报一句话总结，例如：

> ✅ 完整性自检通过，已覆盖 [X] 个角色、[Y] 条核心旅程、[Z] 个异常场景。现在开始生成 PRD。

如果发现遗漏，说明具体遗漏了什么，向用户补问。
