## 2. 产品概述与背景

### 2.1 产品定位

面向个人债券基金投资者的**收益计算工具**。用户输入基金代码，一键获取当前基金的关键收益指标和市场数据。

### 2.2 业务背景

债券基金投资者面临的核心痛点：

1. **数据获取分散**：基金净值、收益率等关键信息散落在不同平台，手动查询耗时
2. **决策缺乏支撑**：看到收益率波动时容易情绪化操作，缺乏客观数据参考
3. **计算门槛高**：七日年化、近1月收益、最大回撤等指标需要跨多个数据源手工计算，本工具自动完成

### 2.3 MVP 核心价值

> **输入基金代码 → 一键算出所有关键收益指标，不用自己翻数据、按计算器。**

### 2.4 MVP 不做什么

- ❌ 策略阈值配置与操作建议生成
- ❌ 定时自动推送（MVP 阶段用户主动触发计算，在屏幕上查看结果）
- ❌ 盘中实时预警
- ❌ 收益趋势图表
- ❌ 推送历史记录
- ❌ 多基金同时监控
- ❌ 日内净值预估算法

## 3. 目标用户

**个人债券基金投资者**

- 持有债券基金，关注核心收益指标
- 需要快速获取基金当前的收益数据，辅助自己判断
- 希望打开工具就能看到关键指标，无需手动四处查数据
- 对技术实现不关心，要求操作简单

## 4. 用户故事与场景

### 4.1 核心用户旅程

#### 旅程：日常查看（高频）

1. 用户打开工具，输入关注的基金代码（如「020741」）
2. 系统展示基金基本信息（名称、类型、最新净值、七日年化等）
3. 用户点击「刷新计算」，系统拉取最新数据并计算收益指标
4. 屏幕展示完整结果

**预期结果**：用户看到：
- 基金名称、代码、类型
- 最新净值与日涨跌幅
- 七日年化、万份收益
- 近1月收益率、近3月最大回撤
- 当前市场利率（10年期国债收益率、信用利差）

### 4.2 用户故事列表（MVP）

| 编号 | 用户故事 | 优先级 |
|------|---------|--------|
| US-01 | 输入基金代码后，系统自动获取该基金的基本信息，以便确认基金正确 | P0 |
| US-02 | 一键触发收益计算，获取最新净值、七日年化、近1月收益等关键指标 | P0 |
| US-03 | 查看完整的计算分析结果，包括基金收益指标和当前市场利率数据 | P0 |

## 5. 功能需求详情

### 5.1 功能清单（MVP）

| 编号 | 功能名称 | 描述 |
|------|---------|------|
| F-01 | 基金信息查询 | 输入基金代码，获取并展示基金基本信息 |
| F-02 | 收益指标计算 | 拉取数据并计算各项收益指标 |

**MVP 闭环流程**：

```
F-01 基金查询 → F-02 收益计算 → 屏幕展示结果
                                    ↑
                          用户下次可点击刷新
```

### 5.2 各功能详细说明

---

#### F-01 基金信息查询

- **功能描述**：用户输入基金代码后，系统调用公开数据接口获取基金基本信息并展示
- **主要流程**：
  1. 用户输入 6 位基金代码
  2. 系统查询基金基本信息
  3. 展示：基金名称、代码、类型（中长期纯债/短债/混合债等）、最新净值、近7日年化、近1月/3月收益率
  4. 用户确认后进入收益计算
- **异常处理**：
  - 代码不存在 → 提示"未找到该基金（代码：XXXXXX），请检查后重新输入"
  - 代码对应非债券型基金 → 提示"该基金为[类型]，当前仅支持债券型基金"
- **关联功能**：F-02（查询后直接进入计算）

---

#### F-02 收益指标计算

- **功能描述**：用户触发后，拉取基金数据和市场数据，计算收益指标体系
- **触发方式**：用户点击「刷新计算」按钮
- **主要流程**：
  1. 用户点击触发计算
  2. 拉取基金已公布最新净值（通常为昨日/T-1 数据）
  3. 拉取基金历史净值序列（近 90 日，用于计算趋势指标）
  4. 拉取当日实时债市指标（10年期国债收益率、信用利差等）
  5. 计算以下指标：

| 指标 | 数据来源 | 计算方式 |
|------|---------|---------|
| 最新单位净值 | 基金公开数据 | 直接获取 |
| 日涨跌幅 | 基金公开数据 | (当日净值 - 前一日净值) / 前一日净值 × 100% |
| 七日年化收益率 | 基金公开数据 | 直接获取或按监管公式计算 |
| 万份收益 | 基金公开数据 | 直接获取 |
| 近1月收益率 | 历史净值计算 | (最新净值 - 30日前净值) / 30日前净值 × 100% |
| 近3月最大回撤 | 历史净值计算 | max(1 - 当日净值 / 区间最高净值) × 100% |
| 10年期国债收益率 | 实时市场数据 | 直接获取 |
| 信用利差（AA+）| 实时市场数据 | AA+企业债收益率 - 同期限国债收益率 |

- **结果格式**：所有收益率、涨跌幅、回撤等指标计算结果统一转换为**百分比格式**（如 0.45% 而非 0.0045），确保用户可直观阅读
- **异常处理**：
  - 数据源部分指标缺失 → 缺失指标标注"N/A"，不影响其他指标计算
  - 数据源完全不可用 → 提示"数据获取失败，请稍后重试"
  - 历史数据不足（如新基金不满 30 天）→ 有多少算多少，不足的指标标"数据不足"
- **关联功能**：被屏幕结果展示依赖

## 6. 业务规则与逻辑边界

### 6.1 核心业务规则

#### BR-01 交易日判定
- 仅在交易日执行计算（排除周末及法定节假日）
- 非交易日用户仍可手动触发，但标注"非交易日，数据可能有延迟"

#### BR-02 数据时效性标注
- 基金净值标注"昨日净值（YYYY-MM-DD）"
- 实时市场数据标注"当前（HH:MM）"

#### BR-03 数据完整性
- 关键数据缺失时标注"N/A"，不中断整体计算
- 历史数据不足时（如新基金不满 30 天），有多少算多少，不足的指标标"数据不足"

### 6.2 边界条件

| 维度 | 约束 | 说明 |
|------|------|------|
| 基金数量 | MVP 阶段 1 只 | 后续版本开放多基金 |
| 数据时间范围 | 历史净值最多取近 90 日 | 超过 90 日对短期决策参考价值有限 |
| 首次使用 | 无历史数据时趋势指标标注"数据积累中" | 已有数据照常展示 |
| 节假日判定 | 以上交所/深交所交易日历为准 | |

## 9. 范围边界

### 9.1 MVP 包含

- ✅ 单只债券基金的信息查询
- ✅ 一键收益指标计算（净值、七日年化、万份收益、近1月收益、最大回撤、市场利率）
- ✅ 屏幕直接展示完整计算结果
- ✅ 架构预留扩展能力

### 9.2 MVP 不包含

| 内容 | 说明 |
|------|------|
| 策略阈值配置 | 不提供加仓/减仓线等自定义阈值功能，MVP 仅做数据计算和展示 |
| 操作建议生成 | 不提供加仓/减仓/持有等投资操作建议，仅展示客观数据供用户自行判断 |
| 定时自动推送 | MVP 阶段用户主动打开工具、点击刷新查看结果 |
| 实时市场预警 | 盘中利率异动监测，后续版本加入 |
| 收益趋势图表 | MVP 以纯文本数值展示，图表后续版本加入 |
| 推送历史记录 | 依赖推送功能，暂不涉及 |
| 日内净值预估 | 算法复杂度高，需要持仓特征数据，MVP 阶段仅展示已公布净值 |
| 多基金同时监控 | MVP 聚焦单基金场景验证核心价值 |
| 多用户体系 | 个人工具起步 |

### 9.3 后续版本规划

| 版本 | 主题 | 内容 |
