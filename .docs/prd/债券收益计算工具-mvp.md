## 2. 产品概述与背景

### 2.1 产品定位

面向个人债券基金投资者的**收益计算与决策辅助工具**。用户输入基金代码、设定策略阈值后，一键获取关键收益指标和操作建议。

### 2.2 业务背景

债券基金投资者面临的核心痛点：

1. **数据获取分散**：基金净值、收益率等关键信息散落在不同平台，手动查询耗时
2. **决策缺乏支撑**：看到收益率波动时容易情绪化操作，缺乏客观指标参考
3. **计算门槛高**：七日年化、近1月收益、最大回撤等指标需要跨多个数据源手工计算

### 2.3 MVP 核心价值

> **输入基金代码 → 设定你的策略阈值 → 一键算出所有关键指标和操作建议，不用自己翻数据、按计算器。**

### 2.4 MVP 不做什么

- ❌ 定时自动推送（MVP 阶段用户主动触发计算，在屏幕上查看结果）
- ❌ 盘中实时预警
- ❌ 收益趋势图表
- ❌ 推送历史记录
- ❌ 多基金同时监控
- ❌ 日内净值预估算法

## 3. 目标用户

**个人债券基金投资者**

- 持有 1 只债券基金，关注核心收益指标
- 有自己的判断逻辑，需要数据支撑而非情绪驱动
- 希望打开工具就能看到关键指标和参考建议
- 对技术实现不关心，要求操作简单

## 4. 用户故事与场景

### 4.1 核心用户旅程

#### 旅程一：首次配置（一次性）

1. 打开工具，输入关注的基金代码（如「020741」）
2. 系统展示基金基本信息（名称、类型、最新净值、七日年化等）
3. 用户确认后进入阈值配置页：
   - 七日年化加仓线（默认 ≥ 3.5%）
   - 七日年化减仓线（默认 ≤ 2.0%）
   - 近1月收益率减仓线（默认 ≤ -0.5%）
   - 最大回撤警戒线（默认 1.0%）
4. 用户可调整或直接使用默认值，保存配置

**预期结果**：配置保存成功，系统展示当前基金的收益概况和首次操作建议

#### 旅程二：日常查看（高频）

1. 用户打开工具，点击「刷新计算」
2. 系统拉取最新数据（基金净值、历史净值序列、实时市场利率）
3. 计算关键收益指标
4. 与用户设定阈值对比，生成操作建议
5. 屏幕展示完整结果

**预期结果**：用户看到：
- 基金名称、代码
- 最新净值与日涨跌幅
- 七日年化、近1月收益率、近3月最大回撤
- 当前市场利率（10年期国债收益率、信用利差）
- 操作建议（加仓 / 减仓 / 持有 / 关注）+ 判断依据

### 4.2 用户故事列表（MVP）

| 编号 | 用户故事 | 优先级 |
|------|---------|--------|
| US-01 | 输入基金代码后，系统自动获取该基金的基本信息，以便确认配置正确 | P0 |
| US-02 | 一键触发收益计算，获取最新净值、收益率等关键指标 | P0 |
| US-03 | 系统基于我预设的阈值规则，自动给出操作建议（加仓/减仓/持有） | P0 |
| US-04 | 自定义收益率阈值，以适应我的投资策略 | P0 |

## 5. 功能需求详情

### 5.1 功能清单（MVP）

| 编号 | 功能名称 | 描述 |
|------|---------|------|
| F-01 | 基金信息查询 | 输入基金代码，获取并展示基金基本信息 |
| F-02 | 策略阈值配置 | 设置收益率阈值等参数（含合理默认值） |
| F-03 | 收益指标计算 | 拉取数据并计算各项收益指标 |
| F-04 | 策略信号生成 | 基于阈值 + 趋势分析生成操作建议 |

**MVP 闭环流程**：

```
F-01 基金查询 → F-02 阈值配置 → F-03 收益计算 → F-04 信号生成 → 屏幕展示结果
                                                                    ↑
                                                          用户下次可点击刷新
```

### 5.2 各功能详细说明

---

#### F-01 基金信息查询

- **功能描述**：用户输入基金代码后，系统调用公开数据接口获取基金基本信息并展示
- **主要流程**：
  1. 用户输入 6 位基金代码
  2. 系统查询基金基本信息
  3. 展示：基金名称、代码、类型（中长期纯债/短债/混合债等）、最新净值、近7日年化、近1月/3月收益率
  4. 用户确认后进入阈值配置
- **异常处理**：
  - 代码不存在 → 提示"未找到该基金（代码：XXXXXX），请检查后重新输入"
  - 代码对应非债券型基金 → 提示"该基金为[类型]，当前仅支持债券型基金"
- **关联功能**：F-02（查询后直接进入配置）

---

#### F-02 策略阈值配置

- **功能描述**：用户设置个性化的策略参数，系统据此生成操作建议
- **主要流程**：
  1. 系统展示所有可配置项及其默认值
  2. 用户修改任意项（也可全部使用默认值直接保存）
  3. 保存配置

- **可配置项及默认值**：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 七日年化 - 加仓线 | ≥ 3.5% | 当七日年化高于此值时建议加仓 |
| 七日年化 - 减仓线 | ≤ 2.0% | 当七日年化低于此值时建议减仓 |
| 近1月收益率 - 减仓线 | ≤ -0.5% | 近1月跌幅超过此值触发关注 |
| 最大回撤警戒线 | 1.0% | 近3月最大回撤超过此值触发预警 |

- **校验规则**：
  - 加仓线必须大于减仓线
  - 百分比阈值 ≥ 0
- **关联功能**：F-01（输入）、F-03、F-04（输出）

---

#### F-03 收益指标计算

- **功能描述**：用户触发后，拉取基金数据和市场数据，计算收益指标体系
- **触发方式**：用户点击「刷新计算」按钮
- **主要流程**：
  1. 用户点击触发计算
  2. 拉取基金已公布最新净值（通常为昨日/T-1 数据）
  3. 拉取基金历史净值序列（近 30 日，用于计算趋势指标）
  4. 拉取当日实时债市指标（10年期国债收益率、信用利差等）
  5. 计算以下指标：

| 指标 | 数据来源 | 计算方式 |
|------|---------|---------|
| 最新单位净值 | 基金公开数据 | 直接获取 |
| 日涨跌幅 | 基金公开数据 | (当日净值 - 前一日净值) / 前一日净值 × 100% |
| 七日年化收益率 | 基金公开数据 | 直接获取或按监管公式计算 |
| 万份收益 | 基金公开数据 | 直接获取 |
| 近1月收益率 | 历史净值计算 | (最新净值 - 30日前净值) / 30日前净值 × 100% |
| 近3月最大回撤 | 历史净值计算 | max(1 - 当日净值 / 区间最高净值) |
| 10年期国债收益率 | 实时市场数据 | 直接获取 |
| 信用利差（AA+）| 实时市场数据 | AA+企业债收益率 - 同期限国债收益率 |

- **异常处理**：
  - 数据源部分指标缺失 → 缺失指标标注"N/A"，不影响其他指标计算
  - 数据源完全不可用 → 提示"数据获取失败，请稍后重试"
  - 历史数据不足（如新基金不满 30 天）→ 有多少算多少，不足的指标标"数据不足"
- **关联功能**：被 F-04 依赖

---

#### F-04 策略信号生成

- **功能描述**：基于用户配置的阈值和计算出的指标，生成操作建议
- **主要流程**：
  1. **阈值检查**：逐一比对当前指标与用户设定的阈值
  2. **趋势分析**：
     - 检查七日年化收益率近 5 日变动方向（持续上行 / 持续下行 / 震荡）
     - 检查净值是否连续 N 日（默认 3 日）下跌
     - 检查当前收益率在近 30 日的分位数位置
  3. **综合判断**，按以下优先级生成信号：

| 信号 | 触发条件 | 优先级 |
|------|---------|--------|
| 🟢 加仓 | 七日年化 ≥ 加仓线，且趋势非持续下行 | 高 |
| 🔴 减仓 | 七日年化 ≤ 减仓线，或近1月跌幅超减仓线，或最大回撤超警戒线 | 最高 |
| 🟡 关注 | 趋势连续下行 ≥ 3 日，或收益率分位进入历史低位 20% | 中 |
| ⚪ 持有 | 所有指标在正常区间，无触发条件 | 低 |

  4. **生成判断依据摘要**：用一句话解释建议理由（如"七日年化 3.8%，高于加仓线 3.5%，近5日趋势上行"）

- **信号优先级规则**：减仓 > 加仓 > 关注 > 持有（风险优先原则）
- **异常处理**：
  - 关键指标缺失 → 降级为"数据不足，建议参考完整信息后手动判断"
- **关联功能**：依赖 F-03

## 6. 业务规则与逻辑边界

### 6.1 核心业务规则

#### BR-01 交易日判定
- 仅在交易日执行计算（排除周末及法定节假日）
- 非交易日用户仍可手动触发，但标注"非交易日，数据可能有延迟"

#### BR-02 数据时效性标注
- 基金净值标注"昨日净值（YYYY-MM-DD）"
- 实时市场数据标注"当前（HH:MM）"

#### BR-03 策略信号优先级
- 减仓 > 加仓 > 关注 > 持有
- 关键数据缺失 → "数据不足，建议手动判断"

#### BR-04 阈值校验
- 加仓线 > 减仓线（必须）
- 百分比阈值 ≥ 0
- 不满足则拒绝保存并提示

### 6.2 边界条件

| 维度 | 约束 | 说明 |
|------|------|------|
| 基金数量 | MVP 阶段 1 只 | 后续版本开放多基金 |
| 数据时间范围 | 历史净值最多取近 90 日 | 超过 90 日对短期决策参考价值有限 |
| 首次使用 | 无历史数据时趋势指标标注"数据积累中" | 阈值对比照常进行 |
| 节假日判定 | 以上交所/深交所交易日历为准 | |

## 7. 验收标准

### F-01 基金信息查询
- ✅ 输入有效债券基金代码（020741），5 秒内展示基金名称、类型、最新净值、七日年化
- ✅ 输入无效代码，提示"未找到该基金"且不进入下一步
- ✅ 输入非债券型基金代码，提示类型不匹配

### F-02 策略阈值配置
- ✅ 加仓线 ≤ 减仓线时拒绝保存并提示"加仓线必须大于减仓线"
- ✅ 所有校验通过后保存成功
- ✅ 新用户首次进入展示默认值，可一键保存

### F-03 收益指标计算
- ✅ 用户点击刷新后，2 分钟内完成所有指标计算
- ✅ 数据源部分不可用时，可计算指标正常展示，缺失指标标注"N/A"

### F-04 策略信号生成
- ✅ 七日年化 ≥ 加仓线且趋势非下行 → 信号为"加仓"
- ✅ 七日年化 ≤ 减仓线 → 信号为"减仓"（优先级最高）
- ✅ 多信号同时触发 → 按减仓 > 加仓 > 关注 > 持有输出
- ✅ 关键数据缺失 → 展示"数据不足，建议手动判断"

### 体验验收
- ✅ 首次配置全流程（基金查询 → 阈值设置 → 首次计算）≤ 10 分钟
- ✅ 日常使用：打开工具 → 点刷新 → 看结果，三步完成
- ✅ 默认阈值可直接使用，无需专业知识即可起步
- ✅ 结果页面信息清晰，一眼看到净值、关键指标、操作建议

## 9. 范围边界

### 9.1 MVP 包含

- ✅ 单只债券基金的信息查询
- ✅ 策略阈值配置（含合理默认值）
- ✅ 一键收益指标计算（净值、七日年化、近1月收益、最大回撤、市场利率）
- ✅ 策略信号生成（加仓/减仓/持有/关注 + 判断依据）
- ✅ 屏幕直接展示完整结果
- ✅ 架构预留多基金扩展能力

### 9.2 MVP 不包含

| 内容 | 说明 |
|------|------|
| 定时自动推送 | MVP 阶段用户主动打开工具、点击刷新查看结果 |
| 实时市场预警 | 盘中利率异动监测，后续版本加入 |
| 收益趋势图表 | MVP 以纯文本数值展示，图表后续版本加入 |
| 推送历史记录 | 依赖推送功能，暂不涉及 |
| 日内净值预估 | 算法复杂度高，需要持仓特征数据，MVP 阶段仅展示已公布净值 |
| 多基金同时监控 | MVP 聚焦单基金场景验证核心价值 |
| 多用户体系 | 个人工具起步 |
