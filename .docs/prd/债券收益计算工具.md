## 2. 产品概述与背景

### 2.1 产品定位

面向个人债券基金投资者的**智能收益追踪与决策辅助工具**。每日自动获取指定债券基金的最新数据，计算关键收益指标，在交易截止（15:00）前推送收益报告与加仓/减仓建议。

### 2.2 业务背景

债券基金投资者面临三个核心痛点：

1. **数据获取分散**：基金净值、收益率、市场利率等关键信息散落在不同平台，每天手动查询耗时且容易遗忘
2. **决策窗口紧张**：基金交易日 15:00 截止申购/赎回，但当日准确净值往往收盘后才公布，投资者在窗口期内缺乏足够数据支撑决策
3. **策略执行依赖自律**：即使有了投资策略（如"收益率跌破 X% 就减仓"），缺少自动监控和提醒机制，容易被情绪左右或错过操作窗口

当前用户通常的做法是：每天手动登录天天基金/蚂蚁财富查看持仓，凭感觉判断是否操作——效率低、一致性差。

### 2.3 产品目标

| 目标 | 衡量标准 |
|------|---------|
| 决策时效 | 每个交易日在 14:30 前完成推送，留给用户 ≥30 分钟操作时间 |
| 数据准确 | 已公布数据与基金公司官方数据一致，估算数据注明"估算值" |
| 策略一致性 | 用户按工具建议操作时，与其预设策略阈值 100% 一致 |
| 使用负担 | 用户日常零操作，仅需首次配置（≤10 分钟）和接收推送 |

### 2.4 核心价值主张

> **每天 3 点前一条推送，用你自己设定的规则替你看盘、算收益、给建议——不用盯盘，不会忘。**

### 3.2 各角色详细画像

**个人投资者（本期核心用户）**

- **使用场景**：每个交易日 14:00-15:00 之间，查看当日推送，决定是否操作基金
- **典型行为特征**：
  - 持有 1-5 只债券基金，关注核心收益指标
  - 有自己的投资策略偏好（偏保守或偏积极）
  - 不一定每天操作，但希望每天了解持仓动态

  - 手动查净值太麻烦，经常忘记

- **期望**：
  - 设好规则后完全自动运行
  - 推送信息一目了然，包含明确的操作建议
  - 出现异常波动时能额外提醒

## 4. 用户故事与场景

### 4.1 核心用户旅程

#### 旅程一：首次配置（一次性）

- **用户目标**：完成工具的初始化配置，让工具开始自动运行
- **前置条件**：用户已部署工具，拥有推送接收端（Bark/PushPlus）
- **操作步骤**：
  1. 打开配置界面，输入关注的基金代码「020741」
  2. 设置推送渠道：选择 Bark 或 PushPlus，填入推送 URL/Token
  3. 配置策略阈值（系统提供默认值，用户可调整）：
     - 七日年化收益率阈值（加仓线 / 减仓线）
     - 近1月收益率阈值
     - 最大回撤警戒线
  4. 设置每日推送时间（默认 14:30）
  5. 保存配置，系统立即执行一次计算并推送测试消息
- **预期结果**：用户收到一条测试推送，内容包含当前该基金的收益概况
- **异常分支**：
  - 基金代码输入错误 → 提示"未找到该基金，请检查代码"
  - 推送 URL 无效 → 发送测试推送失败，提示用户检查配置

#### 旅程二：每日自动推送（日常高频）

- **用户目标**：在下午 3 点前收到当日的收益报告和操作建议
- **前置条件**：已完成配置，当日为交易日，数据源可正常访问
- **操作步骤**：
  1. 系统在每日推送时间（默认 14:30）自动触发计算任务
  2. 拉取基金最新已公布净值（通常为昨日数据）
  3. 拉取当日实时债市行情指标（国债收益率、信用利差等）
  4. 计算关键收益指标，与用户设定阈值对比
  5. 执行趋势分析（近 N 日变动方向、移动均线关系）
  6. 综合生成操作建议（加仓 / 减仓 / 持有 / 关注）
  7. 通过配置的推送渠道发送消息
- **预期结果**：用户手机收到推送，内容包含：
  - 基金名称、代码
  - 最新净值与日涨跌幅
  - 关键收益率指标及与阈值的关系
  - 操作建议 + 判断依据摘要
- **异常分支**：
  - 数据源超时无响应 → 重试 3 次，仍失败则推送"今日数据获取异常，请手动查看"
  - 当日非交易日（周末/节假日）→ 不推送，日志记录"休市日，跳过"
  - 基金处于暂停申购/赎回状态 → 正常推送，但附加"该基金当前暂停交易"提示
  - 推送服务不可用 → 记录日志，次日重试时补推（若数据仍有效）

#### 旅程三：异常波动实时预警（触发式）

- **用户目标**：在交易日盘中，当债市出现剧烈波动时，及时收到预警
- **前置条件**：已完成配置，当日为交易日，已开启实时预警功能（默认开启）
- **操作步骤**：
  1. 系统在交易日 9:30-15:00 期间，每隔 N 分钟（默认 30 分钟）拉取一次实时行情
  2. 当监测到以下任一条件触发时，立即推送预警：
     - 10年期国债收益率单日变动幅度 ≥ X bp（默认 5bp）
     - 信用利差异常走阔 ≥ Y bp（默认 10bp）
     - 央行公开市场操作出现重大信号（净投放/回笼超预期）
  3. 预警推送内容包含：触发指标、变动幅度、对持有债基的预期影响方向
- **预期结果**：用户收到预警推送，可基于市场变化提前判断是否需要在 3 点前操作
- **异常分支**：
  - 盘中数据源偶发不可用 → 跳过本次轮询，下次正常继续
  - 单日多次触发 → 每次触发都推送（不合并），但单日上限 5 条

### 5.2 各功能详细说明

---

#### F-01 基金信息查询

- **优先级**：P0
- **功能描述**：用户输入基金代码后，系统调用公开数据接口获取基金名称、类型、成立日期、规模、费率等基本信息，展示给用户确认
- **前置条件**：有网络连接，数据源可用
- **主要流程**：
  1. 用户输入 6 位基金代码
  2. 系统查询基金基本信息
  3. 展示：基金名称、代码、类型（中长期纯债/短债/混合债等）、最新净值、近7日年化、近1月/3月收益率
  4. 用户确认后进入阈值配置
- **异常处理**：
  - 代码不存在 → 提示"未找到该基金（代码：XXXXXX），请检查后重新输入"
  - 代码对应非债券型基金 → 提示"该基金为[类型]，当前仅支持债券型基金。是否继续关注？"（📝 由用户决定是否扩展支持范围）
- **关联功能**：F-02（查询后直接进入配置）

---

#### F-02 策略阈值配置

- **优先级**：P0
- **功能描述**：用户设置个性化的策略参数，系统据此生成操作建议
- **前置条件**：已成功查询并确认基金
- **主要流程**：
  1. 系统展示所有可配置项及其默认值
  2. 用户修改任意项（也可全部使用默认值直接保存）
  3. 保存配置，系统发送一条测试推送验证推送渠道
- **可配置项及默认值**（📝 默认值基于保守型债基投资策略，可调整）：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 七日年化 - 加仓线 | ≥ 3.5% | 当七日年化高于此值时建议加仓 |
| 七日年化 - 减仓线 | ≤ 2.0% | 当七日年化低于此值时建议减仓 |
| 近1月收益率 - 减仓线 | ≤ -0.5% | 近1月跌幅超过此值触发关注 |
| 最大回撤警戒线 | 1.0% | 近3月最大回撤超过此值触发预警 |
| 每日推送时间 | 14:30 | 建议在 14:00-14:50 之间 |
| 推送渠道 | 待选择 | Bark / PushPlus，需填入 URL 或 Token |
| 实时预警开关 | 开启 | 是否接收盘中异常波动预警 |
| 利率异动阈值 | 5bp | 10年期国债单日变动超过此值触发预警 |

- **异常处理**：
  - 加仓线 ≤ 减仓线 → 校验拦截，提示"加仓线必须大于减仓线"
  - 推送 URL 测试失败 → 提示"推送测试未通过，请检查 URL/Token 是否正确"，允许先保存但标记推送渠道未验证
- **关联功能**：F-01（输入）、F-03、F-04（输出）

---

#### F-03 每日收益计算

- **优先级**：P0
- **功能描述**：每日定时拉取基金数据和市场数据，计算收益指标体系，并基于实时行情**预估当日净值**（在官方净值公布之前，为15:00前决策提供参考）
- **前置条件**：已完成 F-02 配置，当日为交易日
- **主要流程**：
  1. 到达推送时间，触发计算任务
  2. 拉取基金已公布最新净值（通常为昨日/T-1 日数据）
  3. 拉取基金历史净值序列（近 30 日，用于计算趋势指标）
  4. 拉取**基金持仓特征**（有效久期、杠杆率、前5大重仓券 — 基于最近季报）
  5. 拉取当日实时债市指标（30Y/10Y国债收益率及变动、国债期货行情、央行OMO、DR007）
  6. **执行日内净值预估**（详见 [附录A：日内净值预估算法](#附录a日内净值预估算法)）—— 双维度交叉验证，输出预估净值、涨跌幅及置信度
  7. 计算以下指标体系：

| 指标 | 数据来源 | 计算方式 |
|------|---------|---------|
| 最新单位净值 | 基金公开数据 | 直接获取 |
| 日涨跌幅 | 基金公开数据 | (当日净值 - 前一日净值) / 前一日净值 × 100% |
| 七日年化收益率 | 基金公开数据 | 直接获取或按监管公式计算 |
| 万份收益 | 基金公开数据 | 直接获取 |
| 近1月收益率 | 历史净值计算 | (最新净值 - 30日前净值) / 30日前净值 × 100% |
| 近3月最大回撤 | 历史净值计算 | max(1 - 当日净值 / 区间最高净值) |
| 10年期国债收益率 | 实时市场数据 | 直接获取 |
| 30年期国债收益率 | 实时市场数据 | 直接获取 |
| 信用利差（AA+）| 实时市场数据 | AA+企业债收益率 - 同期限国债收益率 |
| 🆕 **日内预估净值** | 算法计算 | 综合久期法 + 期货Beta法双维度交叉验证（详见附录A） |
| 🆕 **日内预估涨跌幅** | 算法计算 | (预估净值 - 昨日净值) / 昨日净值 × 100% |

- **异常处理**：
  - 数据源部分指标缺失 → 缺失指标标注"N/A"，不影响其他指标计算
  - 数据源完全不可用 → 重试 3 次（间隔 30s），仍失败则推送异常通知
  - 历史数据不足（如新基金不满 30 天）→ 可用数据有多少算多少，不够的指标标"数据不足"
- **关联功能**：被 F-04、F-05 依赖

---

#### F-04 策略信号生成

- **优先级**：P0
- **功能描述**：基于用户配置的阈值和计算出的指标，生成操作建议
- **前置条件**：F-03 计算完成
- **主要流程**：
  1. **阈值检查**：逐一比对当前指标与用户设定的阈值
  2. **趋势分析**：
     - 检查七日年化收益率近 5 日变动方向（持续上行 / 持续下行 / 震荡）
     - 检查净值是否连续 N 日（默认 3 日）下跌
     - 检查当前收益率在近 30 日的分位数位置
  3. **综合判断**，按以下优先级生成信号：

| 信号 | 触发条件 | 优先级 |
|------|---------|--------|
| 🟢 加仓 | 七日年化 ≥ 加仓线，且趋势非持续下行 | 高 |
| 🔴 减仓 | 七日年化 ≤ 减仓线，或近1月跌幅超减仓线，或最大回撤超警戒线 | 高 |
| 🟡 关注 | 趋势连续下行 ≥ 3 日，或收益率分位进入历史低位 20% | 中 |
| ⚪ 持有 | 所有指标在正常区间，无触发条件 | 低 |

4. **生成判断依据摘要**：用一句话解释为什么给出这个建议（如"七日年化 3.8%，高于加仓线 3.5%，近5日趋势上行"）

- **异常处理**：
  - 多个信号同时触发 → 按优先级取最高的信号
  - 关键指标缺失导致无法判断 → 降级为"数据不足，建议参考完整信息后手动判断"
- **关联功能**：依赖 F-03，被 F-05 使用

---

#### F-05 每日定时推送

- **优先级**：P0
- **功能描述**：在用户设定的时间，通过所选渠道推送收益报告与操作建议
- **前置条件**：F-03、F-04 执行完成，推送渠道配置有效
- **主要流程**：
  1. 组装推送消息内容
  2. 调用推送服务 API 发送
  3. 记录推送状态（成功/失败/时间）

- **推送消息模板**：

```
📊 华泰保兴安悦债券C（020741）
━━━━━━━━━━━━━━━━━━━━
📅 数据日期：2025-06-23（昨日净值）
⏰ 推送时间：2025-06-24 14:30

💰 净值：1.0234 ｜ 日涨跌：+0.02%
📈 七日年化：3.82%
📉 近1月收益：+0.31%
⚠️  近3月最大回撤：0.45%

🎯 操作建议：🟢 建议加仓
💡 依据：七日年化 3.82% 高于加仓线 3.5%，近5日趋势上行

📡 实时市场（14:00）
10年期国债：2.68%（今日 -1.2bp）
信用利差 AA+：58bp（平稳）
━━━━━━━━━━━━━━━━━━━━
⚙️ 当前规则：加仓≥3.5% | 减仓≤2.0% | 回撤警戒 1.0%
```

- **异常处理**：
  - 推送失败 → 记录日志，3 分钟内重试 1 次；仍失败则标记"推送异常"
  - 推送渠道持续异常（连续 3 次失败）→ 额外推送一条系统告警，提示用户检查推送配置
- **关联功能**：依赖 F-02（配置）、F-03（数据）、F-04（信号）

#### F-06 实时市场预警

- **优先级**：P1
- **功能描述**：在交易日盘中（9:30-15:00）监测利率异动，触发预警时实时推送
- **前置条件**：F-02 中实时预警开关已开启，当日为交易日
- **主要流程**：
  1. 每 30 分钟轮询一次实时市场数据（📝 频率可调整）
  2. 比对当前值与当日基准值（开盘数据）
  3. 触发预警条件时立即推送
  4. 推送内容：触发指标、变动幅度、对持仓债基的预期影响方向

- **预警触发条件**：

| 条件 | 默认阈值 | 说明 |
|------|---------|------|
| 10年期国债收益率单日变动 | ≥ 5bp | 反映利率整体走向 |
| 信用利差（AA+）单日走阔 | ≥ 10bp | 反映信用风险溢价变化 |

  - 单日多次触发 → 每次触发独立推送，单日上限 5 条（超过后不再推送，次日重置）
  - 轮询时数据不可用 → 跳过本次，下次正常继续；连续 3 次不可用 → 推送一条"实时监测暂时中断"通知
- **关联功能**：与 F-05 互补（日常报告 + 异常预警）

## 6. 业务规则与逻辑边界

### 6.1 核心业务规则

#### BR-01 交易日判定

- **规则描述**：仅在交易日执行计算和推送
- **触发条件**：每次到达推送时间时
- **规则逻辑**：根据中国证券市场交易日历判定（排除周末及法定节假日）
- **例外情况**：用户可手动触发一次性计算（不受交易日限制），但推送内容中会标注"非交易日，数据可能有延迟"

#### BR-02 数据时效性标注

- **规则描述**：所有展示的指标必须标注数据所属日期
- **触发条件**：每次推送 / 展示时
- **规则逻辑**：
  - 基金净值数据标注"昨日净值（YYYY-MM-DD）"
  - 实时市场数据标注"实时（HH:MM）"
  - 估算数据明确标注"估算值，仅供参考"
- **例外情况**：无

#### BR-03 策略信号优先级

- **规则描述**：当多个信号条件同时满足时，按固定优先级取最高者
- **触发条件**：F-04 判断过程中
- **规则逻辑**：减仓 > 加仓 > 关注 > 持有
  - 减仓信号优先级最高（宁可错杀不可放过，风险优先原则）
- **例外情况**：关键数据缺失导致无法判断 → 不适用此优先级，直接返回"数据不足"

#### BR-04 阈值校验规则

- **规则描述**：用户设定的阈值必须满足逻辑一致性
- **触发条件**：用户保存配置时
- **规则逻辑**：
  - 加仓线 > 减仓线（必须）
  - 推送时间在 00:00-23:59 之间（格式校验）
  - 百分比阈值 ≥ 0（不能为负）
- **例外情况**：无。不满足则拒绝保存并提示具体错误。

#### BR-05 推送防骚扰

- **规则描述**：避免对用户造成信息轰炸
- **触发条件**：全局推送逻辑中
- **规则逻辑**：
  - 每日定时推送：每交易日 1 条
  - 实时预警：单日上限 5 条
  - 测试推送：配置变更后 1 条
  - 系统异常告警：仅在推送连续失败 3 次时触发
- **例外情况**：用户在短时间内反复修改配置 → 每次保存最多发 1 条测试推送，合并 5 分钟内的修改

## 7. 验收标准

### 7.1 功能验收

**F-01 基金信息查询**
- When 用户输入有效的债券基金代码（020741）, Given 数据源可用, Then 系统在 5 秒内展示该基金的名称、类型、最新净值、七日年化
- When 用户输入无效代码, Given 任意状态, Then 系统提示"未找到该基金"且不进入下一步
- When 用户输入非债券型基金代码, Given 任意状态, Then 系统提示类型不匹配并询问是否继续

**F-02 策略阈值配置**
- When 用户保存配置, Given 加仓线 ≤ 减仓线, Then 系统拒绝保存并提示"加仓线必须大于减仓线"
- When 用户保存配置, Given 所有校验通过, Then 系统保存成功并发出一条测试推送
- When 新用户首次进入配置, Given 无历史配置, Then 所有配置项展示默认值，用户可一键保存

**F-03 每日收益计算**
- When 到达推送时间, Given 为交易日且数据源可用, Then 系统在 2 分钟内完成所有指标计算
- When 数据源部分不可用, Given 部分指标可计算, Then 可计算指标正常展示，缺失指标标注"N/A"

**F-04 策略信号生成**
- When 七日年化 ≥ 加仓线且趋势非下行, Given 数据完整, Then 信号为"加仓"
- When 七日年化 ≤ 减仓线, Given 数据完整, Then 信号为"减仓"，无论其他条件如何
- When 多个信号条件同时触发, Given 数据完整, Then 按减仓 > 加仓 > 关注 > 持有输出最高优先级信号

**F-05 每日定时推送**
- When 到达配置的推送时间, Given 所有前置步骤完成, Then 用户 2 分钟内收到推送，内容包含净值、收益率、建议、依据
- When 推送失败, Given 首次失败, Then 3 分钟内重试 1 次
- When 连续 3 次推送失败, Given 推送渠道可能失效, Then 推送系统告警

**F-06 实时市场预警**
- When 10年期国债单日变动 ≥ 5bp, Given 盘中交易时段, Then 5 分钟内推送预警
- When 单日已推送 5 条预警, Given 再次触发预警条件, Then 不再推送，次日重置计数
- When 连续 3 次轮询失败, Given 数据源不可用, Then 推送"实时监测暂时中断"通知

## 附录A：日内净值预估算法

> **版本**：v1.0 | **最后更新**：2025-06-24  
> **关联功能**：F-03 每日收益计算 — 步骤6「执行日内净值预估」

### A.1 算法背景与业务价值

**核心痛点**：债券基金净值在交易日收盘后公布（通常 20:00 后），但申购/赎回截止时间为 15:00。投资者在决策窗口期内只能看到昨日净值，无法获知今日基金表现。本算法通过实时债市行情数据，在 15:00 前预估当日净值变动方向和幅度，填补决策窗口的信息缺口。

**业务场景**：

```
时间线：
  09:30  债市开盘 → 行情数据开始流动
  14:30  推送触发 → 运行预估算法
  14:50  用户看到推送 → 了解预估净值，做出操作决策
  15:00  交易截止 → 用户完成申购/赎回
  20:00+ 官方净值公布 → 验证预估准确性
```

### A.2 适用条件

| 条件 | 说明 |
|------|------|
| **基金类型** | 纯利率债基金（持仓为国债/政策性金融债，无信用风险敞口） |
| **可用数据** | 基金最近季报（含持仓明细、杠杆率）、实时债市行情 |
| **交易时段** | 交易日 9:30-14:50 |
| **不适用** | 含信用债/可转债的基金、货币基金、权益类基金 |

> 对于持有信用债的基金，需要额外引入信用利差变动因子，算法复杂度显著上升。本版本聚焦纯利率债场景。

### A.3 所需输入参数

#### A.3.1 基金特征参数（首次配置时提取，按季更新）

| 参数 | 单位 | 来源 | 说明 |
|------|------|------|------|
| 持仓券种结构 | — | 基金季报 | 前5大重仓券名称、代码、占比 |
| 综合久期 | 年 | 季报 + 推算 | 按前5大持仓加权平均；基金季报通常直接披露 |
| 有效久期 | 年 | 综合久期 × 杠杆率 | 实际对利率的敏感度 |
| 杠杆率 | 倍数 | 季报 | 债券总资产 / 净资产（例：134% → 1.34） |
| 历史Beta | — | 近30日回归 | Cov(净值涨跌, 30Y期货涨跌) / Var(30Y期货涨跌) |

#### A.3.2 实时市场数据（盘中拉取）

| 数据 | 单位 | 来源 | 频率 | 说明 |
|------|------|------|------|------|
| **30年期国债收益率** | % | 银行间市场 | 实时 | 核心驱动因子，与久期直接关联 |
| **10年期国债收益率** | % | 银行间市场 | 实时 | 组合中10Y品种的定价锚 |
| **30年期国债期货主力合约** | 元 | 中金所 | 实时 | 用于Beta法计算及方向验证 |
| 央行公开市场操作 | 亿元 | 央行公告 | 盘中 | 净投放/回笼规模（含MLF到期续作） |
| 资金利率 DR007 | % | 银行间市场 | 实时 | 反映流动性松紧 |
| 2年期/5年期国债收益率 | % | 银行间市场 | 实时 | 辅助判断曲线形态变化 |

### A.4 核心算法：双维度交叉验证

#### 方法一：久期法（Duration Method）

**原理**：债券价格变动 ≈ -久期 × 收益率变动。通过持仓加权久期和实时收益率变动，推算净值变动。

**公式**：

```
净值变动(%) ≈ -有效久期 × ΔY_30Y + (-10Y久期敞口 × ΔY_10Y) + 日内票息

其中：
  有效久期 = 组合久期 × 杠杆率
  ΔY_30Y  = 当前30Y收益率 - 昨收30Y收益率（单位：%，如 +0.75bp = +0.0075%）
  10Y久期敞口 = 组合中10Y品种占比 × 10Y久期 × 杠杆率
  日内票息 ≈ 组合加权利率 / 365（微调项，通常 +0.005%/日 量级）
```

**实例**（020741 华泰保兴安悦债券C）：

```
已知：
  组合久期 ≈ 20年（对标30Y国债）
  杠杆率 ≈ 1.34
  有效久期 ≈ 20 × 1.34 ≈ 26.8年
  30Y收益率上行 0.75bp = 0.0075%
  10Y收益率下行 0.15bp = -0.0015%（正面贡献，但量级小）

计算：
  30Y冲击 = -26.8 × 0.000075 = -0.201%
  10Y贡献 = -7.2 × (-0.000015) ≈ +0.011%
  日内票息 ≈ +0.005%
  ────────────────────
  合计 ≈ -0.185%  → 约 -0.11% ~ -0.18%（考虑持仓并非100%对标30Y，取折中）
```

**注意事项**：
- 该方法对久期估算精度敏感。若季报后基金经理调仓，实际久期可能与估算偏离
- 收益率曲线非平行移动时，需按关键期限拆分计算（如30Y上行但10Y下行）

#### 方法二：期货Beta法（Futures Beta Method）

**原理**：利用30年期国债期货与基金净值的历史相关性，通过期货实时涨跌推算净值变动。

**公式**：

```
净值变动(%) ≈ Beta × 期货涨跌幅(%)

其中：
  Beta = Cov(近30日净值涨跌序列, 近30日30Y期货涨跌序列) / Var(近30日30Y期货涨跌序列)
  例：020741 近30日 Beta ≈ 0.56
```

**实例**（020741）：

```
今日30Y期货盘中约 -0.08%
净值变动 ≈ 0.56 × (-0.08%) = -0.045%  → 约 -0.04%
```

**注意事项**：
- Beta 是非平稳参数，市场情绪剧烈变化时可能漂移
- 期货反映的是预期而非已实现利率，偶尔与现货收益率走势背离

### A.5 情景区间

除点估计外，输出三种情景的概率区间，帮助用户理解不确定性：

| 情景 | 含义 | 概率 | 典型分布 |
|------|------|------|---------|
| **乐观** | 尾盘利率回落，净值修复 | 25-30% | 约 +0.5 标准差 |
| **基准** | 当前趋势延续至收盘 | 45-55% | 等于点估计值 |
| **悲观** | 尾盘抛压加大，净值扩大跌幅 | 15-20% | 约 -1.0 标准差 |

概率赋权基于：
- 当前处于交易日的哪个阶段（早盘更不确定，14:30 后接近收盘更确定）
- 资金面松紧方向（宽松 → 乐观增权；紧张 → 悲观增权）
- 期货基差状态（期货较现货溢价 → 方向可能转好）

### A.8 误差来源与校准

| 误差来源 | 影响程度 | 降低策略 |
|----------|---------|---------|
| 基金持仓滞后（季报最多滞后3个月） | ⭐⭐⭐ 高 | 推送标注"基于XX季报持仓"；季报更新后3日内重新校准 |
| 盘中收益率→收盘有差异 | ⭐⭐ 中 | 标注数据时间戳（HH:MM）；14:30后推送时差异通常较小 |
| 基金经理主动调仓 | ⭐⭐⭐ 高 | 无法避免；通过跟踪误差积累发现异常时告警 |
| 杠杆率变动（申赎导致） | ⭐⭐ 中 | 季报更新时重新计算；规模大幅变动时额外关注 |
| 政金债-国债利差变动 | ⭐ 低 | 关注利差日频变动，必要时在久期法中增加利差因子 |
| Beta参数漂移 | ⭐ 低 | Beta使用30日滚动窗口在线更新 |

**校准机制**：每次官方净值公布后，回溯比对预估值与实际值：

```
校准指标：
  - 方向准确率（预估方向 vs 实际方向）
  - 绝对误差（|预估涨跌幅 - 实际涨跌幅|，单位 bp）
  - 方向错误时的偏差放大系数

目标：
  - 方向准确率 > 70%
  - 绝对误差 < 8bp（区间：1σ范围内）
  - 无系统性高估/低估（均值回归检验通过）

若连续5个交易日方向错误 → 触发模型告警，排查久期/Beta参数是否失效
```

### A.9 局限性声明

1. **本算法输出是"估算"而非"预测"**：基于已知持仓和当前行情的数据联动，不涉及对未来走势的预判
2. **本算法不预测债市方向**：不试图预测利率涨跌，仅反映"如果当前行情持续到收盘，净值大约是多少"
3. **不适用于剧烈波动日**：当30Y收益率单日变动超过5bp时，置信度自动降至"低"，因为尾盘可能出现方向反转
4. **算法不能替代基金公司官方数据**：用户决策应综合参考预估和官方数据
5. **持仓数据存在时滞**：基金经理的主动操作在季报发布前无法获知，实际久期可能已偏离估算值
