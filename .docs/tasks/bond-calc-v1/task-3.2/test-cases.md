### TC-014: 非债券型基金 — 橙色警告样式
- **类型**: 功能测试
- **关联验收标准**: SUB-3.2.2-3
- **前置条件**: FundPreview 已挂载
- **输入**:
  - fundInfo = `{ ..., fund_type: "混合偏债", ... }`
- **执行步骤**:
  1. 传入非债券型 FundInfo（fund_type 不含"债"字，但 API 返回了 TYPE_MISMATCH？不，这是另一种场景 — fund_type 含"债"但非纯债券型）
  2. 实际上，根据 store 的 `isBondFund` 计算逻辑：`fund_type.includes('债')` 才为 true
  3. 传入 fund_type = "混合型"（不含"债"）
- **预期输出**:
  - 显示橙色/amber 警告样式（边框、背景或文字颜色变化）
  - 显示类型不匹配提示文本，如"该基金不是债券型基金"
- **清理**: 无

> **说明**：`isBondFund` 的计算逻辑是 `fund_type?.includes('债')`。因此"混合偏债"（含"债"）仍视为债券型。真正触发警告的是如"混合型"、"股票型"等不含"债"的类型。如果 API 返回 40002 错误，则在 FundInput 层面已拦截，不会传递到 FundPreview。FundPreview 的警告样式覆盖的是：API 返回成功但基金类型并非债券型（如未来数据源扩展）的边界场景。
